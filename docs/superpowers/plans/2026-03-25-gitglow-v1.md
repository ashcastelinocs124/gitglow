# Gitglow V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first working version of Gitglow: a GitHub profile README generator with GitHub OAuth, guided storytelling input, preview, hosted visual assets, manual export, and optional auto-sync to the user's profile README repository.

**Architecture:** Build a Next.js app with a Postgres-backed data model, GitHub OAuth via Auth.js, a background job layer for imports and asset refreshes, and a markdown generation pipeline that separates structured profile modeling from prose and asset output. Keep the README markdown GitHub-compatible and use hosted generated images for visual richness.

**Tech Stack:** Next.js App Router, TypeScript, Tailwind CSS, Prisma, PostgreSQL, Auth.js, Vitest, Testing Library, Playwright, Zod, GitHub REST/GraphQL APIs, `react-markdown`, `remark-gfm`, `satori` or chart/image generation library for hosted assets

---

## Scope Split

This spec spans multiple subsystems, but they form one coherent v1 product if implemented in this order:

1. App foundation and data model
2. Auth and GitHub import
3. Storytelling questionnaire and featured project selection
4. Structured profile model and README generation
5. Hosted asset generation and refresh jobs
6. Preview, export, and sync
7. History, rollback, and reliability hardening
